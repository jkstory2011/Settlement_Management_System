- 화주사 "구간별 계약 단가표"(`/shippers/[id]/rates`) 재설계:
  - 기존에는 CJ 원본 기본운임 **금액**(예: 1,750원)을 직접 입력/선택하는 방식이었는데, 같은 금액이 항상 같은 타입(극소/소/중 등)을 의미하지 않는다는 게 확인돼서 폐기.
  - 지금은 CJ대한통운 실제 타입(**극소/소/중/대1/대2/이형/취급제한**) 7종 고정 드롭다운 + 화주사와 실제 계약한 단가를 입력하는 순수 참고표로 변경. 택배사 청구 금액과 무관하며, 원본 내역서를 계약단가로 수정할 때 참고용.
  - DB 스키마 변경: `shipper_rate_tiers.cj_base_fee`(numeric) → `cj_type`(text).
  - 이 변경으로 인해 업로드/재계산 시 금액 기반 자동 단가 매칭 로직(`shipper-match.js`, `recompute_batch_applied_amounts_chunk`)은 제거함 — 현재는 원본 총운임(total_fee)을 그대로 적용. **"화주사/단가 재계산" 버튼을 눌러도 에러는 안 나지만, 계약단가가 자동 반영되지는 않는 상태.**
